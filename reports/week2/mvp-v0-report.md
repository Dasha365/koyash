# MVP v0 Report

**Назначение и описание фундамента:**
Запускаемый технический фундамент бэкенда для проекта KOYASH. Реализован на FastAPI, обеспечивает автоматическую генерацию Swagger-документации и настроен на работу с базой данных MongoDB Atlas для предоставления списка товаров и их rule-based фильтрации.

**Deployment URL / ссылка на артефакт:**
https://koyash-production.up.railway.app/docs

**Ссылка на публичное видео:**
https://youtu.be/ftTVnoXQvI8

**Связь с прототипом и историями MVP v1:**
- `GET /products` и `POST /recommend` предоставляют данные каталога и реализуют базовую rule-based логику рекомендаций, которая в дальнейшем будет привязана к истории US-04 (формирование косметички по фильтрам).

**Текущие ограничения, заглушки, моки:**
- Отсутствует интеграция с LLM (рекомендации пока работают только на жёстких фильтрах базы: segment, vegan, cruelty_free, allergens_norm).
- Не реализована система авторизации и сохранения анкет пользователей.
- `GET /products` возвращает все 69 товаров без пагинации (ограничение MVP v0).

**Ссылка на локальную инструкцию:**
См. README.md в корне репозитория.

**Сценарий smoke-check:**
- **Доступ:** открыть http://localhost:8000/docs или Deployment URL.
- **Шаги:**
  1. Выполнить `docker compose --env-file backend/.env up --build` в корне репозитория.
  2. Сделать запрос `GET /health` через Swagger UI.
  3. Сделать запрос `GET /products` через Swagger UI.
- **Ожидаемо:** контейнеры поднимаются без ошибок; `/health` отдаёт статус 200; `/docs` успешно грузится; `/products` возвращает JSON с реальными документами из коллекции MongoDB Atlas.
